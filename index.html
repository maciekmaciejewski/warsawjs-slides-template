Dlaczego twój administartor Cię nienawidzi?

14-06-2017

@kalafiorLoose

Jesteś użytkownikiem/użytkowniczką zaawansowanym.

  1. Więc zachowuj się jak użytkownik/użytkowniczka zaawansowany/a.
  2. Znaj narzędzia, których używasz.
  3. Szanuj swoje klucze.
  4. "Nie działa" jest mało precyzyjne.
  5. Nie bądź gamoniem.

Ponieważ z szufladki z gamoniami jest bardzo trudno się wydostać.

Nie proś o deploye pod koniec dnia, a już na pewno nie w piątek.

  1. Rano/w poniedziałek będę bardzo niezadowolony z twojej postawy.
  2. Inni też.
  3. Zrobię rollback. I będę niezadowlony. Inni na pewno też.
  4. Skoro do mnie dzwonią, że twój wspaniały kod nie działa, zgadnij do kogo ja zadzwonie, żeby go naprawił.
  5. My git commit && git push are so much bigger than yours.

A tera skup się...

https://warsawjs.com:443/index.html?event=warsawjs34#prezntacja2

  1. Adres IP, DNS oraz inne części Internetów.
  2. Serwer www.
  3. Kody odpowiedzi http.

Poznaj swój stack.

  1. Nie będziesz musiał/a wywarzać otwartych drzwi.
  2. Zrozum swój git-flow.

Nie bądź jak złota rybka.

  1. Brew install, apt install, setup.exe.
  2. Wersje wszystkiego.
  3. Nie hardkoduj, ale jak już się nie możesz powstrzymać, to mi o tym powiedz.
  4. Najlepiej to prowadź dokumentacje.
  5. Szanuj swoje środowisko developerskie.

Nie testuj sam/a swojego kodu.

  1. Przecież skoro go napisałaś/eś to wiesz, że działa, bo sprawdziłaś/eś.
  2. Tylko, że nie.
  3. Pisz dokumentacje.
  4. Pisz testy.
  5. Nazywaj commity po ludzku.

Nie:

  1. Dostaniesz roota na produkcji.
  2. Nie zrobisz tego małego INSERTA na produkcyjnej bazie danych.
  3. Nie poprawie tak na szybko tej literówki na serwerze produkcyjnym.

Problemy z wydajnością możemy rozwiązać tylko razem.

  1. Dodanie setki gigabajtów RAMu, 30 procesorów, superszybkich dysków SSD, 50 kolejnych VPSów, i nowej lokacji AWSa w Warszawie brzmi dużo lepiej, kiedy mówisz to temu, kto za to zapłaci.
  2. Skalowanie w góre jest trudne i kosztowne, a pozatym w końcu zabraknie slotów na ten RAM, więc pisz tak, żeby się dało skalować bok.

Nie zawsze zależymy do siebie. Zarówno Ty jak i ja

  1. Wymogi bezpieczeństwa.
  2. Wymogi licencyjne.
  3. Compatibility lists.
  4. Umowy.
